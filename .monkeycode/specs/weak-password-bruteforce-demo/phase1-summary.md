## 1. 项目概述

实现一个在 Windows 系统（Windows 7/10/11）中运行的弱口令枚举暴力破解演示程序。用户输入目标密码后，程序通过 3 个工作线程并发枚举尝试，直到匹配成功或用户手动终止。

### 原始需求

1. 用户输入目标密码
2. 按 5 种规则顺序枚举（纯数字、字母头+数字、数字+字母尾、小写混合、全字母混合）
3. 3 个进程/线程同时枚举，任一找到即终止
4. 显示进度、累计用时，支持提前终止
5. 破解完成后显示总尝试次数和时间

## 2. 技术选型与关键决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 编程语言 | Python 3.13 | 利用 Free-threading 无 GIL 特性 |
| 并发方案 | `threading` 多线程 | 3.13 free-threading 模式下可真正并行，共享状态简单 |
| 进程间通信 | 原生内存共享 | 线程共享同一进程内存，无需序列化 |
| 包管理器 | uv | 快速、现代，原生支持 free-threading Python |
| 打包工具 | PyInstaller | 生成独立 exe，无需用户安装 Python |
| UI 架构 | 引擎与 UI 解耦 | `UICallback` 协议接口，便于扩展 Web UI |

### 为什么选择 Python 3.13 Free-threading？

Python 3.13 提供了 experimental free-threading 构建（通过 `python3.13t` 访问），GIL 被移除，多线程可以真正并行执行 CPU 密集型任务。相比多进程方案的优势：
- 线程间共享状态更简单，无需 `multiprocessing.Value/Array` 等复杂 API
- 内存开销更低（无需为每个进程复制内存）
- 启动速度更快（无需 spawn 新进程）

## 4. 模块说明

| 模块文件 | 职责 |
|----------|------|
| `brute_force/engine.py` | 核心引擎：任务生命周期管理、回调调度、状态汇总 |
| `brute_force/thread_manager.py` | 线程管理：创建/监控/终止工作线程 |
| `brute_force/shared_state.py` | 共享状态：`threading.Lock` 保护的线程间共享数据 |
| `brute_force/worker.py` | 工作线程：枚举循环、密码比对、批量更新尝试次数 |
| `brute_force/enum_rules.py` | 枚举规则：5 种规则的迭代器生成器 |
| `brute_force/ui_interface.py` | UI 接口：`UICallback` Protocol 定义 |
| `brute_force/cli.py` | CLI 实现：控制台输入、进度刷新、Ctrl+C 处理 |
| `brute_force/web_api.py` | Web 预留：REST API + WebSocket 框架（注释标记） |
| `main.py` | 程序入口 |

## 5. 枚举规则

| 规则编号 | 描述 | 字符集 | 长度 | 数量级 |
|---------|------|--------|------|--------|
| 1 | 纯数字 | 0-9 | 1-8 | ~10^8 |
| 2 | 字母开头+数字 | a-zA-Z + 0-9 | 2-9 | ~5.7*10^9 |
| 3 | 数字+字母结尾 | 0-9 + a-zA-Z | 2-9 | ~5.7*10^9 |
| 4 | 数字+小写混合 | 0-9, a-z | 1-8 | ~2.9*10^12 |
| 5 | 数字+大小写混合 | 0-9, a-z, A-Z | 1-8 | ~2.2*10^14 |

### 线程分配策略

按轮询方式分配规则到 3 个线程：
- Worker 0: 规则 1（纯数字）、规则 4（数字+小写混合）
- Worker 1: 规则 2（字母头+数字）、规则 5（全混合）
- Worker 2: 规则 3（数字+字母尾）

## 6. 性能优化

- **批量更新**：工作线程本地计数累加 10000 次后才写入共享状态，减少锁竞争
- **字符集预计算**：DIGITS、LOWERCASE、UPPERCASE 等字符集在模块加载时预计算
- **早期终止**：发现密码后立即设置 `found` 标志，其他线程检查后退出

## 7. 构建与打包

### 开发环境

```bash
# 安装 Python 3.13 free-threading
uv python install 3.13t

# 运行程序
uv run --python 3.13t main.py

# 运行测试
uv run --python 3.13t tests/test_enum_rules.py
uv run --python 3.13t tests/test_shared_state.py
uv run --python 3.13t tests/test_engine.py
```

### Windows 打包

在 Windows 上执行：
```cmd
build.bat        # 使用 python3.13t + pip
build.ps1        # 使用 uv（推荐）
```

生成文件：`dist\WeakPasswordBruteForce.exe`

## 10. 注意事项

- 本程序仅为演示用途，请勿用于非法目的
- 规则 5（全字母混合）空间极大，8 位密码可能需要极长时间
- 建议在测试时使用短密码（1-4 位）
- 当前环境 Linux 上打包生成的是 ELF 格式可执行文件，Windows 打包需在 Windows 上执行
