# 弱口令枚举暴力破解演示程序 - 需求文档

## 功能需求

### FR-1: 用户输入
**当** 用户启动程序或上一次破解完成/被终止时，**系统** 提示用户输入目标密码，**以便** 作为后续枚举比对的基准。

### FR-2: 枚举尝试
**当** 用户输入目标密码后，**系统** 按以下顺序枚举密码并比对：
- **FR-2.1**: 1位到8位纯数字
- **FR-2.2**: 1位英文字母（先小写后大写）开头，后接1位到8位数字
- **FR-2.3**: 1位到8位数字，以1位字母结尾
- **FR-2.4**: 1位到8位数字与小写英文字母混合
- **FR-2.5**: 1位到8位数字与大小写英文字母混合

**当** 枚举密码与目标密码一致时，**系统** 记录尝试次数和时间，保存密码。

### FR-3: 多进程并发
**当** 开始枚举时，**系统** 启动三个进程同时执行枚举任务，**当** 任一进程找到正确密码时，**系统** 输出结果并终止所有进程。

### FR-4: 进度显示
**当** 破解进行时，**系统** 持续显示：
- 各进程当前尝试进度
- 累计尝试次数
- 本次破解累计用时

### FR-5: 结果输出
**当** 破解成功时，**系统** 显示总尝试次数和总用时。

### FR-6: 提前终止
**当** 破解进行时，用户可触发提前终止，**系统** 终止所有进程并显示截止时的用时。

### FR-7: 输入提示
**当** 等待用户输入时，**系统** 显示清晰的输入提示信息。

## 非功能需求

### NFR-1: 兼容性
程序须在 Windows 7、Windows 10、Windows 11 上正确运行。

### NFR-2: 性能
枚举过程应高效执行，充分利用多进程并发能力。

### NFR-3: 独立性
程序为独立可执行文件，无需额外运行时依赖。
